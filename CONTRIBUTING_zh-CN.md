# 为 RequestHelper 做贡献

[English](./CONTRIBUTING.md) | 中文

感谢您有兴趣为 RequestHelper 做贡献！我们欢迎来自社区的贡献。

> **⚠️ 重要：本项目完全由 AI 编写，我们要求所有贡献也必须使用 AI 生成的代码。**

## 🤖 AI 生成代码要求

所有对本项目的代码贡献**必须由 AI 助手生成**，例如：

- GitHub Copilot
- ChatGPT (OpenAI)
- Claude (Anthropic)
- 其他 AI 编程助手

### 为什么只接受 AI 生成的代码？

本项目是一个 AI 代码生成能力的实验和展示。为了保持项目的一致性和实验性质，我们要求所有贡献都使用 AI 辅助工具生成。这有助于：

- 展示 AI 在实际项目开发中的能力
- 保持整个代码库的代码风格一致性
- 作为 AI 辅助开发工作流程的参考案例
- 探索 AI 生成软件的极限和可能性

## 📋 贡献指南

### 开始之前

1. **检查现有 issue**：查找与您的贡献相关的现有 issue 或讨论
2. **先开 issue**：对于重大更改，请先开 issue 讨论您的提议
3. **使用 AI 工具**：确保您可以访问 AI 编程助手

### 进行贡献

1. **Fork 仓库**并从 `master` 创建新分支
2. **使用 AI 生成代码**：使用您首选的 AI 编程助手生成代码
3. **遵循项目约定**：参考项目的编码模式和架构
4. **测试您的更改**：
   - 更改后运行 `npm run build`
   - 使用 `test/test-page.html` 进行手动测试
   - 验证扩展在 Chrome 中正确加载
5. **记录 AI 使用情况**：在 Pull Request 描述中说明：
   - 使用了哪些 AI 工具
   - 使用 AI 的提示词或方法
   - 进行的任何手动调整（如适用）

### Pull Request 流程

1. **更新文档**：如果您的更改影响面向用户的功能，请更新相关文档
2. **遵循提交约定**：编写清晰、描述性的提交消息
3. **提交 PR**：创建一个 Pull Request，清楚描述：
   - 进行了哪些更改
   - 为什么需要这些更改
   - 使用了哪个 AI 工具
   - 如何测试这些更改
4. **回应反馈**：回应代码审查意见并进行必要的调整

## 🛠️ 开发设置

1. 克隆您的 fork：
   ```bash
   git clone https://github.com/your-username/request-helper.git
   cd request-helper
   ```

2. 安装依赖：
   ```bash
   npm install
   ```

3. 构建扩展：
   ```bash
   npm run build
   ```

4. 在 Chrome 中加载：
   - 打开 `chrome://extensions/`
   - 启用"开发者模式"
   - 点击"加载已解压的扩展程序"
   - 选择 `dist/` 文件夹

## 🧪 测试

- 需要手动测试（目前没有自动化测试套件）
- 使用 `test/test-page.html` 触发各种请求场景
- 在 Chrome DevTools 中测试以验证捕获和拦截
- 在 `chrome://extensions/` 检查后台 service worker 日志

## 📝 代码风格

- 遵循项目中现有的代码模式
- 后台脚本使用 ES6 模块
- UI 组件使用原生 JavaScript（无框架）
- 遵循 viewer 和 options 页面的模块化架构模式
- 为复杂逻辑添加适当的注释

## 🔍 项目架构

贡献之前请熟悉项目架构：

- **双层拦截**：Content script + 页面注入脚本
- **模块系统**：后台使用 ES6 模块，UI 使用 script 标签
- **存储架构**：Chrome storage API，具有特定的 key 模式
- **规则系统**：URL 模式匹配，支持捕获/阻断/修改操作

详细架构文档请参见 `IMPLEMENTATION.md` 和内联注释。

## 📮 报告问题

报告问题时，请包含以下内容：

> **注意**：由于本项目是完全由 AI 驱动的项目，所有 issue 将由 AI 助手处理。虽然我们努力解决所有问题，但不保证一定能够解决。感谢您的理解与耐心。

### 应包含的内容

- **描述**：清楚描述问题
- **重现步骤**：详细的重现问题的步骤
- **期望行为**：您期望发生什么
- **实际行为**：实际发生了什么
- **环境**：Chrome 版本、操作系统、扩展版本
- **截图/日志**：如适用，包含截图或控制台日志

### Issue 类型

- **Bug 报告**：某些功能不能正常工作
- **功能请求**：新功能或改进的建议
- **问题**：关于使用或实现的问题
- **文档**：文档的改进或更正

## 🎯 贡献领域

我们特别欢迎以下领域的贡献：

- **Bug 修复**：修复报告的问题或您发现的 bug

- **UI/UX 增强**：改进用户界面或体验
- **新的过滤选项**：为查看器添加新的过滤功能

- **文档**：改进文档、添加示例或翻译
- **测试用例**：向 `test/test-page.html` 添加更多测试场景
