### Before You Start

1. **Check existing issues**: Look for existing issues or discussions related to your contribution
2. **Open an issue first**: For major changes, please open an issue to discuss your proposal before starting work
3. **Use AI tools**: Ensure you have access to an AI coding assistant

### Making Contributions

1. **Fork the repository** and create a new branch from `master`
2. **Generate code using AI**: Use your preferred AI coding assistant to generate the code
3. **Follow project conventions**: Refer to the project's coding patterns and architecture
4. **Test your changes**:
   - Run `npm run build` after making changes
   - Test manually using `test/test-page.html`
   - Verify the extension loads correctly in Chrome
5. **Document AI usage**: In your Pull Request description, specify:
   - Which AI tool(s) you used
   - The prompts or approach you used with the AI

1. **Update documentation**: If your changes affect user-facing features, update relevant documentation
2. **Follow commit conventions**: Write clear, descriptive commit messages
3. **Submit PR**: Create a Pull Request with a clear description of:
   - What changes were made
   - Why the changes are needed
   - Which AI tool was used
   - How the changes were tested
4. **Address feedback**: Respond to code review comments and make necessary adjustments

## üõ†Ô∏è Development Setup

1. Clone your fork:
   ```bash
   git clone https://github.com/your-username/request-helper.git
   cd request-helper
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Build the extension:
   ```bash
   npm run build
   ```

4. Load in Chrome:
   - Open `chrome://extensions/`
   - Enable "Developer mode"
   - Click "Load unpacked"
   - Select the `dist/` folder

## üìÆ Reporting Issues

When reporting issues, please:

> **Note**: As this is an entirely AI-driven project, issues will be processed by AI assistants. While we strive to address all concerns, resolution is not guaranteed. We appreciate your understanding and patience.

### What to Include

- **Description**: Clear description of the issue
- **Steps to reproduce**: Detailed steps to reproduce the problem
- **Expected behavior**: What you expected to happen
- **Actual behavior**: What actually happened
- **Environment**: Chrome version, OS, extension version
- **Screenshots/logs**: If applicable, include screenshots or console logs

### Issue Types

- **Bug reports**: Something isn't working correctly
- **Feature requests**: Suggestions for new features or improvements
- **Questions**: Questions about usage or implementation
- **Documentation**: Improvements or corrections to documentation
